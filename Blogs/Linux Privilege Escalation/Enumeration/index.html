Linux Privilege Escalation Enumeration

When you gain access to a computer system, whether through an exploit or by using a low-privileged account, the next step is enumeration. This process involves gathering important details about the system. It helps you understand what kind of access you currently have and what vulnerabilities or misconfigurations you might be able to exploit to gain higher-level privileges. Enumeration is a crucial step, whether you're trying to break into a system or simply assessing your current access level after successfully exploiting a vulnerability.

Commands Used in Enumeration

Here are some common Linux commands that can help you gather information about the system and identify potential vectors for privilege escalation (gaining higher privileges):

1. hostname

The hostname command simply returns the name of the system (or host) you're connected to. The hostname can sometimes give clues about the system’s role, location, or environment, which might help you decide on your next steps in an attack or penetration test.

2. uname -a

The uname -a command prints detailed system information, including the operating system's kernel version, which can be useful for identifying any known vulnerabilities specific to that version. If there are security flaws in the kernel, you might be able to exploit them to gain root access.

3. /proc/version

The /proc/version file contains version details about the system’s kernel and may also include information about the system’s compiler (like GCC). The /proc directory in general is a special virtual filesystem in Linux that provides details about the system and running processes.

This information can help you identify the exact version of the kernel running on the system, which is important for finding known vulnerabilities that can be exploited.

4. /etc/issue

The /etc/issue file typically contains a brief message or system information about the operating system version and some other details. It's displayed before the login prompt in many Linux distributions. However, this information can be easily modified, so it's not always reliable. Still, it can give you useful clues.

5. ps

The ps command shows you a list of processes running on the system. By default, it only shows processes for the current user’s session. This command is useful for finding out what software is running, whether there are any unusual processes, and if there are any background tasks that could potentially be exploited.

The output of ps includes:

  • PID: Process ID (a unique number identifying each running process)
  • TTY: The terminal or session where the process is running
  • TIME: The amount of CPU time used by the process (not the time the process has been running)
  • CMD: The name of the command that started the process

6. env

The env command lists environmental variables. These variables can contain useful system information, such as the paths to important files, user-specific data, and configurations. This is a great way to find potential clues about the system’s environment that could lead to privilege escalation.

7. sudo -l

The sudo -l command shows you a list of commands that your user account is allowed to run with root privileges (or other higher privileges). If you have sudo access, this information is vital, as it can help you determine if there are any commands you can run that could lead to privilege escalation, such as running a shell or executing dangerous commands as root.

8. ls

The ls command is used to list files in a directory. To gather more detailed information, you should use the -la options with ls to display hidden files and detailed file information such as ownership and permissions.

Hidden files (those beginning with a dot .) might contain sensitive data or misconfigurations that can help in privilege escalation.

9. id

The id command shows the user ID (UID), group ID (GID), and the groups the user belongs to. This is important for understanding your current privilege level on the system. Additionally, you can use the id command to check the privilege and group membership of other users on the system.

10. /etc/passwd

The /etc/passwd file contains user account information for the system. Each line represents a different user, and it includes details such as the user’s name, UID, GID, and home directory. By reading this file, you can easily find out who the users on the system are and gather additional information for your next steps.

11. history

The history command displays a list of recent commands that have been run in the current terminal session. By looking through this history, you might uncover useful information such as user actions, mistakes, or potentially even usernames or passwords stored in the command history. It’s a good idea to examine this output for any sensitive data that might have been left behind.

Why Enumeration is Important

In summary, enumeration is an essential first step in understanding a system after you’ve gained initial access. By using these commands, you can gather crucial information that helps you assess your current privileges and identify possible ways to escalate those privileges. Always be thorough, as every detail could reveal a new opportunity or vector for privilege escalation.
